# Models

This folder holds the model configuration and metrics. The trained weight files
(`*.pt`, ~1.7 GB total) are too large to bundle and are regenerated by running
the notebook (`notebook/Rahul_Mishra_453_P4_Notebook.ipynb`).

## Included

- `model_config.json` — label ordering, party-encoding map, metadata dimension,
  max sequence length, and headline metrics for every model.
- `results.json` — full accuracy / macro-F1 / weighted-F1 for all models.

## Trained weights (regenerate via the notebook)

| File | Size | Description |
| --- | --- | --- |
| `bert_text_only.pt` | 418 MB | Best LIAR model (macro-F1 0.284) |
| `bert_metadata_fusion.pt` | 419 MB | Dual-stream text + metadata |
| `roberta_text_only.pt` | 476 MB | RoBERTa variant |
| `bert_welfake_binary.pt` | 418 MB | WELFake binary classifier (98.8%) |

The notebook uses a load-or-train pattern: if a `.pt` file is present it is
loaded; otherwise the model is trained and saved. The web application backend
(`webapp/backend`) loads `bert_text_only.pt` and `bert_metadata_fusion.pt`.
